Perceptron

The perceptron is the simplest neural network unit, acting as a binary linear classifier. It learns to separate two classes by adjusting weights.

How it works:

  • Input Layer: Receives feature values (blue nodes)
  • Weights: Each input has an adjustable weight
  • Summation: Weighted sum of inputs plus bias
  • Activation: Step function produces binary output (red/green)
  • Learning: Adjusts weights based on classification errors
Input Features Output Decision

Restricted Boltzmann Machine (RBM)

An RBM is a generative neural network with two layers: visible and hidden. It learns to represent data by finding patterns in the input.

How it works:

  • Visible Layer: Receives input data (blue nodes)
  • Hidden Layer: Learns hidden features (purple nodes)
  • Bidirectional: Energy flows both ways during training
  • No connections: Within the same layer (restricted)
Visible Layer Hidden Layer

Autoencoder

An autoencoder compresses data into a lower-dimensional representation, then reconstructs it back to the original form.

How it works:

  • Input Layer: Original data (green nodes)
  • Encoder: Compresses to latent space (yellow nodes)
  • Latent Space: Compressed representation (orange nodes)
  • Decoder: Reconstructs from latent space (yellow nodes)
  • Output Layer: Reconstructed data (green nodes)
Input Encoder Latent Decoder Output

Ising Model

A statistical physics model describing magnetic spin interactions on a lattice. Demonstrates phase transitions and emergent collective behavior.

How it works:

  • Spins: Each cell has +1 (up/white) or -1 (down/black)
  • Neighbors: Spins interact with adjacent cells
  • Energy: Lower when aligned neighbors
  • Temperature: Controls randomness and phase transitions
  • Dynamics: Monte Carlo updates minimize energy

Hopfield Network

A recurrent neural network that serves as content-addressable memory, storing and retrieving patterns through energy minimization.

How it works:

  • Fully Connected: Every neuron connects to every other
  • Symmetric Weights: Connection strength is bidirectional
  • Pattern Storage: Memories stored in weight matrix
  • Recall: Noisy input converges to stored pattern
  • Energy Landscape: Patterns are stable attractors

Deep Perceptron (Multi-Layer Perceptron)

A feed-forward neural network with one or more hidden layers, capable of learning complex non-linear decision boundaries.

How it works:

  • Input Layer: Receives feature vectors
  • Hidden Layers: Multiple layers learn hierarchical features
  • Activation Functions: Non-linear transformations (ReLU, sigmoid)
  • Output Layer: Classification or regression output
  • Backpropagation: Gradient-based weight optimization

Normalizing Flow

A sequence of invertible transformations that map a simple distribution to a complex one, enabling exact likelihood computation.

How it works:

  • Base Distribution: Simple Gaussian starting point
  • Invertible Transforms: Bijective mappings (coupling layers)
  • Jacobian: Tracks volume changes for likelihood
  • Forward Pass: Generate samples from complex distribution
  • Inverse Pass: Compute exact log-likelihood

Bidirectional CNN Encoder-Decoder

A convolutional architecture that encodes spatial information and decodes it for tasks like segmentation and image-to-image translation.

How it works:

  • Conv Encoder: Downsamples with pooling to capture features
  • Skip Connections: Preserve spatial information across scales
  • Bottleneck: Compressed feature representation
  • Conv Decoder: Upsamples to reconstruct spatial dimensions
  • Applications: Segmentation, super-resolution, style transfer

Mamba2 Architecture

A state-space model with selective gating mechanisms that efficiently processes long sequences with linear complexity, rivaling Transformers.

How it works:

  • State Space: Continuous-time dynamics for sequences
  • Selective Scan: Input-dependent state transitions
  • Gating: Control what information flows through time
  • Linear Complexity: O(n) vs O(n¬≤) for attention
  • Long Context: Efficiently handles very long sequences

CUDA Programming Model

GPU parallel computing with NVIDIA CUDA: visualize thread hierarchies, memory types, and massively parallel execution.

How it works:

  • Grid: Collection of thread blocks
  • Blocks: Groups of threads executed together
  • Threads: Individual execution units (thousands in parallel)
